# 获取上下文技能

## 功能说明

通过远程 API 获取系统当前的上下文环境信息，返回当前会话、当前模块、当前选择的若干对象
等运行时环境数据，供后续操作使用或直接展示给用户。

## 前置条件

- 无需额外参数，可直接调用
- 若用户在特定对象上下文中查询（如想了解某对象的环境），可附带 `uid` 参数

## 执行步骤

### 第一步：调用上下文接口

使用 `httpGet` 工具，请求 Teamcenter Rich Client MicroService 的完整 URL。必须使用配置替换后的 URL，不要读取环境变量、不要扫描本机端口。

- path：`{TEAMCENTER_RICH_CLIENT_MICROSERVICE_URL}/context`
- queryParams：（无，留空）

若用户指定了某对象的上下文，则：

- path：`{TEAMCENTER_RICH_CLIENT_MICROSERVICE_URL}/context`
- queryParams：`uid={uid}`

### 第二步：处理返回结果

根据 `httpGet` 工具返回的 status 值，按下表响应用户：

| status | 含义       | 响应用户                              |
| ------ | ---------- | ------------------------------------- |
| 200    | 获取成功   | ✅ 将返回的上下文信息整理后展示给用户 |
| 401    | 未授权     | ❌ 当前未登录或会话已过期，请重新登录 |
| 500    | 服务器错误 | ❌ 服务器异常，请稍后重试             |
| error  | 网络异常   | ❌ 无法连接服务器，请确认服务是否启动 |

### 第三步：展示上下文信息

请求成功后（status 200），将返回数据按以下方式呈现：

1. 若返回字段可识别（如 `uid`、`user`、`session`、`env` 等），以结构化方式列出
2. 若字段未知，原样展示 JSON 内容
3. 若后续操作需要上下文中的某个字段，主动告知用户可直接引用

## 安全要求

- 禁止将用户的任何其他输入拼接到 path 或 queryParams 中
- `uid` 只能来自用户明确提供的值，不得推断或伪造
