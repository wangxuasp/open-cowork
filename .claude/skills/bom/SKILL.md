# BOM 操作技能

## 支持的操作

| 操作             | 说明                               |
| ---------------- | ---------------------------------- |
| 1、创建单层 BOM  | 为指定父级节点建立一组子级节点关系 |
| 2、查询 BOM 组成 | 根据输入查询指定BOM的组成          |

---

## 操作一：创建单层 BOM

### 功能说明

在父级节点下批量挂载子级节点，建立单层产品结构（装配关系）。
一次调用只建立**一层**父子关系，多层结构需多次调用。

### 前置条件

执行前必须收集以下信息：

| 参数       | 是否必填 | 说明                              |
| ---------- | -------- | --------------------------------- |
| `parent`   | ✅ 必填  | 父级节点的 UID                    |
| `children` | ✅ 必填  | 子级节点 UID 的集合，至少提供一个 |

若用户未提供，按以下顺序询问：

1. 「请提供父级节点的 UID」
2. 「请提供子级节点的 UID，多个请用逗号分隔」
3. 获得对象的 UID（格式示例：`gXhVErqOZ_jnSD`）
4. UID 必须符合标准 UUID 格式，否则拒绝执行并重新询问

### 执行步骤

#### 第一步：构造请求体

将收集到的参数组装为 JSON：

```json
{
  "parent": "{parent_uid}",
  "children": ["{child_uid_1}", "{child_uid_2}"]
}
```

#### 第二步：调用创建接口

使用 `httpPost` 工具，{TEAMCENTER_RICH_CLIENT_MICROSERVICE_URL}由实际访问URL替换传入以下参数：

- path：`{TEAMCENTER_RICH_CLIENT_MICROSERVICE_URL}/bom/create/single`
- body：第一步构造的 JSON 对象

#### 第三步：处理返回结果

根据返回的 status 值响应用户：

| status | 含义       | 响应用户                                                  |
| ------ | ---------- | --------------------------------------------------------- |
| 200    | 创建成功   | ✅ 单层 BOM 创建成功，父级 `{parent}` 已挂载 {n} 个子节点 |
| 400    | 参数错误   | ❌ 请求参数有误，请检查 UID 格式是否正确                  |
| 404    | 节点不存在 | ❌ 父级或子级节点不存在，请确认 UID 是否有效              |
| 409    | 关系已存在 | ⚠️ 部分或全部子节点已在该 BOM 下，无需重复创建            |
| 500    | 服务器错误 | ❌ 服务器异常，请稍后重试                                 |
| error  | 网络异常   | ❌ 无法连接服务器，请确认服务是否启动                     |

# 操作二：查询 BOM 组成

### 功能说明

根据输入的根节点uid，使用`itkHttpGet`工具查询该节点下的BOM组成内容，返回所有子级节点的uid、name、type等信息。

### 前置条件

执行前必须收集以下信息：

| 参数  | 是否必填 | 说明         |
| ----- | -------- | ------------ |
| `uid` | ✅ 必填  | 根节点的 UID |

若用户未提供，按以下顺序询问：

1. 获得对象的 UID（格式示例：`gXhVErqOZ_jnSD`）
   2UID 必须符合标准 UUID 格式，否则拒绝执行并重新询问

### 执行步骤

#### 第一步：根据 UID 构造查询零组件ID

使用mcp tools去获取uid对应的零组件ID

#### 第二步：调用查询bom的接口

使用 `itkHttpGet` 工具，{item_id}由第一部结果产生，{BASE_URL}由实际访问URL替换传入以下参数：

- path：`{BASE_URL}/bom?item_id={item_id}`

#### 第三步：处理返回结果

根据返回的 status 值响应用户：

| status | 含义       | 响应用户                                 |
| ------ | ---------- | ---------------------------------------- |
| 200    | 过去成功   | ✅ BOM 获取成功                          |
| 400    | 参数错误   | ❌ 请求参数有误，请检查 UID 格式是否正确 |
| 500    | 服务器错误 | ❌ 服务器异常，请稍后重试                |
| error  | 网络异常   | ❌ 无法连接服务器，请确认服务是否启动    |

---

## 安全要求

- `parent` 和 `children` 的值只能来自用户明确提供的 UID，不得推断或伪造
- 禁止将其他用户输入拼接到请求体中
- `children` 必须为数组格式，即使只有一个子节点也需包装为数组：`["uid"]`
