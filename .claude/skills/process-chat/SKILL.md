# 工艺问答技能

## 功能说明

通过配置的知识库 HTTP API 发送用户问题，并将接口返回的回答整理后展示给用户。

## 接口配置

- **请求地址**: `{KNOWLEDGE_BASE_HTTP_URL}`
- **请求路径**: `/api/chat`
- **请求方式**: HTTP POST
- **请求头**: `Content-Type: application/json`
- **请求体**:

```json
{
  "question": "工艺问题",
  "keywords": ["关键字1", "关键字2"],
  "includeDebug": true
}
```

## 执行步骤

### 第一步：确认问题正文

从用户消息中提取要提问的内容，填入 `question` 字段。

如果用户只表达了想查询知识库，但没有给出明确问题，应先询问用户要查询的问题正文，不要编造默认问题。

### 第二步：调用 Qdrant Chat 接口

使用可发送 HTTP POST JSON 请求的工具访问配置地址：

- url：请求地址+请求路径
- body：参考接口json样例

优先使用`python`生成本地脚本执行再执行，脚本参考${SKILL_DIR}\references\query_rag.py
不要将问题拼接到 URL 或 query string 中；必须放在 JSON body 的 `question` 字段里。

### 第三步：响应

处理时应：

1. 持续读取响应片段，按返回顺序合并内容。
2. 如果响应片段是 JSON，优先提取可识别的回答字段，如 `answer`、`content`、`message`、`text`、`delta`。
3. 如果字段未知，保留原始片段并整理为可读文本。
4. 流结束后，将完整回答展示给用户。

### 第四步：错误处理

根据 HTTP 状态或工具错误给出清晰说明：

| 状态     | 含义         | 响应方式                                             |
| -------- | ------------ | ---------------------------------------------------- |
| 200      | 请求成功     | 整理并展示完整回答                                   |
| 400/422  | 请求参数错误 | 提示问题正文或 JSON 格式可能不符合接口要求           |
| 500      | 服务端错误   | 提示 Qdrant Chat 服务处理失败                        |
| 网络错误 | 无法连接     | 提示确认知识库 HTTP URL 配置是否正确、服务是否已启动 |

## 安全要求

- 只访问配置替换后的 `{KNOWLEDGE_BASE_HTTP_URL}` 地址。
- 不要向请求中加入用户未明确提供的敏感信息、凭证或本地文件内容。
- 不要改变 `stream` 的值；除非用户明确要求，否则保持 `true`。

## 示例

用户：`帮我问知识库：整机工艺路线是什么？`

请求体：

```json
{
  "question": "工艺问题",
  "keywords": ["关键字1", "关键字2"],
  "includeDebug": true
}
```

返回后：将接口回答整理为自然语言回复用户。
