# 打开技能

## 功能说明

通过远程 API 将指定对象发送给系统默认模块打开，效果类似操作系统的"打开方式"，
由服务端决定用哪个模块来呈现该对象。

## 前置条件

- 必须获得对象的 UID（格式示例：`wiilZfO$Z_jnSD`）
- 如用户未提供 UID，先询问：「请提供要打开的对象 UID」

## 执行步骤

### 第一步：调用发送接口

使用 `httpGet` 工具，传入以下参数（将 `{uid}` 替换为实际值）：

- path：`{TEAMCENTER_RICH_CLIENT_MICROSERVICE_URL}/sendto`
- queryParams：`uid={uid}`

### 第二步：处理返回结果

根据 `httpGet` 工具返回的 status 值，按下表响应用户：

| status | 含义       | 响应用户                              |
| ------ | ---------- | ------------------------------------- |
| 200    | 打开成功   | ✅ 对象 {uid} 已由系统默认模块打开    |
| 404    | 对象不存在 | ❌ 未找到 UID 为 {uid} 的对象         |
| 500    | 服务器错误 | ❌ 服务器异常，请稍后重试             |
| error  | 网络异常   | ❌ 无法连接服务器，请确认服务是否启动 |

## 安全要求

- 只将 `{uid}` 替换为用户提供的值，其余参数保持不变
- 禁止将其他任何用户输入拼接到 queryParams 中
