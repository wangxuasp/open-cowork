# Teamcenter 数据集操作技能

本技能通过调用远程 API，完成 Teamcenter 数据集的创建、文件导入、删除、文件替换等操作。

## API 基础信息

- **GET 请求**: `{TEAMCENTER_RICH_CLIENT_MICROSERVICE_URL}/dataset?action={action}&uid={uid}[&其他参数]`
- **POST 请求**: `{TEAMCENTER_RICH_CLIENT_MICROSERVICE_URL}/{path}`，参数以 JSON 格式放入请求体
- **BASE_URL**: 由用户提供，或从上下文中获取httpGet, httpPost工具已经定义

---

## 支持的操作

| 编号 | 操作       | 方法 | 说明                                                   |
| ---- | ---------- | ---- | ------------------------------------------------------ |
| 1    | 创建数据集 | POST | 在指定容器下新建数据集对象                             |
| 2    | 导入文件   | GET  | 导入文件并创建数据集。此时不需要执行创建数据集的操作。 |
| 3    | 删除数据集 | GET  | 删除指定数据集及其关联文件                             |
| 4    | 替换文件   | POST | 用新文件替换数据集中的已有文件                         |

---

## 1：创建数据集（create）

**用途**：在 Teamcenter 指定容器（Item、文件夹等）下新建一个数据集对象。

**请求方式**：POST

**请求地址**：`{BASE_URL}/dataset/create`

**请求体**：

```json
{
  "uid": "{容器对象的 UID}",
  "name": "{数据集名称}",
  "type": "{数据集类型}"
}
```

**参数说明**：

| 参数   | 必填 | 说明                                                                          |
| ------ | ---- | ----------------------------------------------------------------------------- |
| `uid`  | 是   | 父容器对象的 UID（如 Item Revision 的 UID）                                   |
| `name` | 是   | 数据集名称                                                                    |
| `type` | 否   | 数据集类型，如 `UGMASTER`、`DirectModel`、`Text` 等，不填由服务端使用默认类型 |

**示例**：

```json
{
  "uid": "wiilZfO$Z_jnSD",
  "name": "设计图纸_v1",
  "type": "DirectModel"
}
```

**注意**：创建成功后，返回结果中的 `datasetUid` 是后续导入文件操作的必要参数，请保存。

---

## 2：导入文件（import）

**用途**：使用`httpGet`工具导入文件并创建数据集。此时不需要执行创建数据集的操作。

**请求方式**：GET

**请求地址**：`{BASE_URL}?action=import&filePath={filePath}&parent={uid}`

**请求体**：无

**参数说明**：

| 参数       | 必填 | 说明                                                  |
| ---------- | ---- | ----------------------------------------------------- |
| `filePath` | 是   | 文件名，必须包含扩展名和路径，如 `c:/123/drawing.pdf` |
| `parent`   | 否   | 数据集的父级对象uid                                   |

---

## 3：删除数据集（delete）

**用途**：删除指定数据集及其所有关联文件，操作不可逆。

**请求方式**：GET

**请求格式**：

```
GET {BASE_URL}?action=delete&uid={datasetUid}
```

**参数说明**：

| 参数     | 必填 | 说明               |
| -------- | ---- | ------------------ |
| `action` | 是   | 固定值：`delete`   |
| `uid`    | 是   | 要删除的数据集 UID |

**示例**：

```
GET https://api.example.com/dataset?action=delete&uid=TbcDEf$X_kpQR
```

**⚠️ 注意**：删除操作不可逆，且会同时删除数据集下的所有关联文件。执行前**必须向用户二次确认**：

> 「即将删除数据集 `{uid}`，此操作不可恢复，是否确认继续？」

---

## 4：替换文件（replace）

**用途**：用新文件替换数据集中已有的文件，保持数据集 UID 不变。

**请求方式**：POST

**请求地址**：`{BASE_URL}/dataset/replace`

**请求体**：

```json
{
  "uid": "{数据集的 UID}",
  "fileName": "{要替换的目标文件名，含扩展名}",
  "fileContent": "{新文件内容，Base64 编码}"
}
```

**参数说明**：

| 参数          | 必填 | 说明                                         |
| ------------- | ---- | -------------------------------------------- |
| `uid`         | 是   | 数据集的 UID                                 |
| `fileName`    | 是   | 被替换的文件名（需与数据集中已有文件名一致） |
| `fileContent` | 是   | 新文件内容的 Base64 编码字符串               |

**示例**：

```json
{
  "uid": "TbcDEf$X_kpQR",
  "fileName": "report.pdf",
  "fileContent": "JVBERi0xLjUKJ..."
}
```

**注意**：`fileName` 必须与数据集中已存在的文件名完全匹配，否则视为新增而非替换。

## 5：导出文件（export）

**用途**：使用`httpGet`工具导出文件到本地路径。

**请求方式**：GET

**请求地址**：`{BASE_URL}?action=export&uid={uid}`

**请求体**：无

**参数说明**：

| 参数  | 必填 | 说明        |
| ----- | ---- | ----------- |
| `uid` | 是   | 数据集的uid |

## 通用错误处理

| status | 含义       | 响应用户                                        |
| ------ | ---------- | ----------------------------------------------- |
| 200    | 操作成功   | ✅ 按各操作的成功提示响应                       |
| 400    | 参数错误   | ❌ 请求参数有误，请检查 UID 或文件名格式        |
| 404    | 对象不存在 | ❌ 未找到指定数据集或容器，请确认 UID 是否正确  |
| 409    | 冲突       | ⚠️ 数据集名称已存在或文件已被锁定，请确认后重试 |
| 500    | 服务器错误 | ❌ 服务器异常，请稍后重试                       |
| error  | 网络异常   | ❌ 无法连接服务器，请确认服务是否启动           |

---

## 参数缺失时的处理

| 缺失参数        | 操作               | 处理方式                         |
| --------------- | ------------------ | -------------------------------- |
| `BASE_URL`      | 所有               | 询问用户 API 地址                |
| `uid`（容器）   | 创建               | 询问：「请提供父容器对象的 UID」 |
| `uid`（数据集） | 导入 / 删除 / 替换 | 询问：「请提供数据集的 UID」     |
| `filePath`      | 导入 / 替换        | 询问：「请提供文件完整路径」     |

---

## 安全要求

- 所有 UID、文件名只能来自用户明确提供的值，不得推断或伪造
- 删除操作必须经过用户二次确认，严禁自动执行
- 禁止将其他用户输入拼接到请求路径或请求体中
